## 模块1：风机-集电线路功率关系分析

📂 目录：[`01_风机集电线路功率关系分析/`](01_风机集电线路功率关系分析/)

📄 分析报告：[分析报告_风机集电线路全站功率层级关系.md](01_风机集电线路功率关系分析/分析报告_风机集电线路全站功率层级关系.md)

### 核心问题

- 风机 SCADA 汇总功率 与 集电线路 CT 测点功率 之间的关系是什么？
- 三条集电线路（丙/丁/戊）的传输损耗有何差异？为什么？
- 风机不发电时的厂用电从哪里来？

### 脚本说明

| 脚本 | 功能 | 输出 |
|------|------|------|
| `提取场站集电线路-全站有功.py` | 从原始 SCADA CSV 提取所需列，计算风机有功汇总（S1/S2策略）| `DATA/峡阳B/#7-1*_with_sum*.csv` |
| `功率层级关系分析.py` | EDA 主分析：时序、散点、差值分布、分功率区间对比 | 图 01-08, `analysis_report.txt` |
| `功率层级关系分析-补充图表.py` | 传输损耗深度分析（分线路对比、月度变化、消耗状态分析）| 图 15-24 |

### 主要结论

| 线路 | 总体传输损耗率 | 特点 |
|------|-------------|------|
| 丙（BING）| ~3.5% | 中等，明阳风机 47 台 |
| 丁（DING）| ~2.4% | **最低**，东气风机 43 台，损耗最稳定 |
| 戊（WU）  | ~6.2% | **最高**，明阳风机 47 台，箱变铁损贡献大 |

## 模块2：数据异常分析

📂 目录：[`02_数据异常分析/`](02_数据异常分析/)

📄 分析报告：[分析报告_数据质量分析_风机SCADA连续相同数据异常分类与清洗建议.md](02_数据异常分析/分析报告_数据质量分析_风机SCADA连续相同数据异常分类与清洗建议.md)

### 核心问题

风机 SCADA 数据中存在**连续多个时间戳五元组（STATUS, ACTIVE_POWER, REACTIVE_POWER, WINDSPEED, WINDDIRECTION）完全相同**的现象，需判断原因并制定清洗策略。

### 三类异常分类框架

根据**同一分钟内同时开始重复的风机数量**：

| 类型 | 判断条件 | 物理含义 | 清洗建议 |
|------|---------|---------|---------|
| **第一类**：全场通讯中断 | ≥ 80 台同时开始重复 | SCADA 上层系统整体中断（如定时轮询）| 删除 |
| **第二类**：部分通讯中断 | 5~79 台同时开始重复 | 集电线路级/网段级通讯节点故障 | 删除 |
| **第三类**：单机异常 | < 5 台同时开始 | 单台风机通讯故障/运维停机/数据漂移 | 按子类处理 |

### 脚本说明

| 脚本 | 功能 | 输出位置 |
|------|------|---------|
| `检查集电线路数据连续相同情况.py` | 检测集电线路各字段连续相同段 | `分析结果/集电线路重复检测/峡阳B/` |
| `检查风机数据连续相同情况.py` | 批量检测 137 台风机五元组联合重复 | `分析结果/风机重复检测/峡阳B/` |
| `数据质量分析-连续相同异常类型分类与清洗建议.py` | 对重复段按三类框架分类，生成清洗建议 | `分析结果/数据质量分析/峡阳B/`，图 25-30 |

### 分类统计（峡阳B，16,258 条重复段）

| 异常类型 | 段数 | 总记录数 | 建议 |
|---------|------|--------|------|
| 第一类-全场通讯中断 | 11,189 | 125,220 | **删除** |
| 第二类-部分通讯中断 | 1,442 | 329,514 | **删除** |
| 第三类-单机非零卡值 | 1,068 | 248,285 | **删除** |
| 第三类-单机通讯故障（零值）| 643 | 936,166 | **删除** |
| 第三类-发电状态零值 | 218 | 481,026 | **删除** |
| 正常停机 | 1,438 | 181,969 | **保留** |
| 零值-状态待核实 | 260 | 26,167 | **人工复核** |
