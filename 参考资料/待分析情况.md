# 待分析情况



每台风机包含5类数据，以63号风机为例，包括'STATUS_#63', 'ACTIVE_POWER_#63', 'REACTIVE_POWER_#63', 'WINDSPEED_#63', 'WINDDIRECTION_#63'，分别是状态码、有功、无功、风速、风向。

但是从时序数据分析来看，每台风机的这5类数据存在连续时间内保持不变的情况，我不知道这些数据是什么情况，是不是通信链路断了，重复上一个时刻的数据，还是其他的什么情况。我已经将所有风机连续时间相同的重复值组合、开始时间、结束时间、持续长度统计出来了，就是“联合重复值检测结果.CSV”文件。同时，集电线路数据也存在连续时间保持不变的情况。

“DATA/峡阳B”文件夹中有多个按照“参考资料/广东场站功率计算公式.md”计算的文件。“参考资料/风机状态吗说明.md”文件中包含了峡阳B场站三种不同厂商风机的状态码说明。

我前期做了一些分析，初步总结了三类异常，但是不确定总结的准不准确，可以给你提供一个思路。

| **异常类型** | **描述**                                                     | **原因初判**               | **涉及范围** |
| ------------ | ------------------------------------------------------------ | -------------------------- | ------------ |
| 第一类       | **所有**风机在同一时间段内出现数据重复，持续时长较短。       | 通讯链路整体断开           | 全场         |
| 第二类       | **部分**风机在同一时间段内出现数据重复，持续时长较长。       | 通讯链路部分断开           | 多台风机     |
| 第三类       | **单台**风机在连续时间段内数据重复，状态码可能为发电/无通讯/运维等，持续时长最长。 | 运维操作、设备掉线或假数据 | 单台风机     |

我初步认为连续相同的大于0的功率，那肯定是异常的，因为功率不可能一直保持不变。但是很多时候遇到的情况是功率小于等于0，这我就不知道风机确实是没有运行，还是说风机正常运行但通信链路断开了，导致数据一直不变。我想的方法是看看这时候利用集电线路上风机功率之和与集电线路功率做比较，看看是不是存在明显数据差值，你有没有其他方法呢
