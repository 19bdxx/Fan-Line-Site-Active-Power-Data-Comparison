# 模块2：数据异常分析

本模块针对风机 SCADA 数据中出现的**连续相同数据（卡值/数据冻结）**现象，进行检测、分类和清洗建议。

## 三类异常框架

| 类型 | 判断条件 | 物理含义 |
|------|---------|---------|
| **第一类**：全场通讯中断 | ≥ 80 台风机同时开始重复 | SCADA 系统整体通讯中断（如定时轮询任务）|
| **第二类**：部分通讯中断 | 5~79 台风机同时开始重复 | 集电线路级或网段级通讯节点故障 |
| **第三类**：单机异常 | < 5 台同时开始 | 单台风机通讯故障 / 运维停机 / 数据漂移 |

## 文件说明

| 文件 | 说明 |
|------|------|
| `检查集电线路数据连续相同情况.py` | **步骤1**：检测集电线路各字段连续相同段，输出到 `分析结果/集电线路重复检测/` |
| `检查风机数据连续相同情况.py` | **步骤2**：批量检测 137 台风机五元组联合重复段，输出到 `分析结果/风机重复检测/` |
| `数据质量分析-连续相同异常类型分类与清洗建议.py` | **步骤3**：按三类框架分类，生成清洗建议 CSV（`分析结果/数据质量分析/`）及图25-30 |
| `分析报告_数据质量分析_风机SCADA连续相同数据异常分类与清洗建议.md` | 完整分析报告 |

## 运行顺序

```bash
python 检查集电线路数据连续相同情况.py
python 检查风机数据连续相同情况.py
python 数据质量分析-连续相同异常类型分类与清洗建议.py
```
