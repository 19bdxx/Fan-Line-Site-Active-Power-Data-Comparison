## 1. 背景与核心问题

### 问题描述

`freeze_count_segments_v3.csv` 记录了每条集电线路上出现"风机 SCADA 数据冻结"（连续重复）的时间段。每个分段有三个关键字段：

- **`BING/DING/WU重复风机功率之和MW`**：该分段内冻结风机的有功功率之和（0 ~ 300 MW 不等）
- **`BING/DING/WU异常占比`**：该时间段内对应集电线路 CT 损耗模型判异的分钟数占比（0 ~ 1）
- **`持续时长(min)`**：分段持续时间

用户观察到这三个指标呈现截然不同的组合，无法简单地"全部删除"或"全部保留"。

### 核心逻辑

集电线路损耗模型使用 `L = FAN_SUM − CT`，其中：

```
FAN_SUM = Σ（所有本线路风机的有功功率）
CT      = 集电线路 CT 表测量值
L       = 传输损耗（正常应 > 0）
```

风机数据冻结对 `L` 的影响取决于**冻结时风机的功率值**：

| 冻结功率 | 对 FAN_SUM 的影响 | 对 L 的影响 |
|:---:|:---:|:---:|
| **= 0 MW** | 停机风机贡献为 0，FAN_SUM 中该部分本来就是 0 | **几乎无直接影响** |
| **> 0 MW（较小）** | FAN_SUM 被锁定在某个小值，与实际偏差小 | **影响可忽略** |
| **> 0 MW（较大）** | FAN_SUM 被锁定在错误的高/低值，随时间积累误差 | **影响显著，必须清除** |

`CT异常占比` 反映的是**集电线路 CT 侧**的数据质量，而非直接由风机冻结造成。当冻结风机功率为 0 时，CT 异常占比高说明 CT 本身存在问题（传感器偏差、负损耗等）。

## 2. 两个关键维度的含义

### 维度 1：`{LINE}重复风机功率之和MW`（记为 `P_freeze`）

- **P_freeze = 0**：冻结风机均处于停机或零功率状态。FAN_SUM 中这些风机的贡献本来就是零，数据冻结对损耗计算**无直接影响**。此时 `L` 的质量完全取决于 CT 的可靠性。
  
- **0 < P_freeze < 1 MW**：冻结功率极小（通常为单台风机，约 0.979 MW 对应 WU 线路某台定点冻结风机）。相对于线路总装机容量（WU ≈ 376~470 MW），这不足装机的 0.3%，造成的 FAN_SUM 误差可忽略不计。

- **1 ≤ P_freeze ≤ 5 MW**：冻结功率属于较低范围（约 2~10 台停机边缘的风机）。对损耗的影响属于中等，需结合 CT 异常占比综合判断。

- **P_freeze > 5 MW**：冻结功率显著。线路总损耗通常在 2~10 MW 量级，>5 MW 的冻结误差已与损耗本身同量级，**必须剔除**。

### 维度 2：`{LINE}异常占比`（记为 `r_anom`）

- **r_anom ≤ 10%**：分段持续时间内仅有极少数时刻 CT 损耗被判异，可认为 CT 数据基本可信。
  
- **10% < r_anom ≤ 20%**：CT 有轻度异常，属于背景噪声水平，通常由短时的负损耗或量程波动引起，总体仍可保留。

- **20% < r_anom ≤ 30%**：CT 出现中等程度异常，属于"灰色地带"，需要结合**分段时长**判断：
  - 短时（≤ 60 min）：大概率是短暂的传感器抖动，可保留
  - 长时（> 8 小时）：可能是 CT 传感器持续故障，建议删除或标记

- **r_anom > 30%**：该时段超过 30% 的时刻 CT 损耗被判异，损耗数据失真严重，**建议删除**。

## 3. 决策框架：五类分段标签

基于上述两维度，将每条线路的分段划分为 **5 类**：

```
                        异常占比 (r_anom)
                    ≤ 10%     10%~20%    20%~30%    > 30%
                ┌──────────┬──────────┬──────────┬──────────┐
功率=0          │  ✅ 保留  │  ✅ 保留  │  ⚠️ 灰色  │  ❌ 删除  │
                ├──────────┴──────────┴──────────┴──────────┤
0 < P < 1 MW    │              ✅ 保留（功率可忽略）            │
                ├──────────┬──────────┬──────────┬──────────┤
1 ≤ P ≤ 5 MW   │  ✅ 保留  │  ✅ 保留  │  ❌ 删除  │  ❌ 删除  │
                ├──────────┴──────────┴──────────┴──────────┤
P > 5 MW        │              ❌ 删除（功率影响显著）           │
                └─────────────────────────────────────────────┘
```

### 五类标签定义

| 标签 | 条件 | 建议 | 原因 |
|:---:|:---|:---:|:---|
| **🟢 A 类** | P_freeze = 0 **且** r_anom ≤ 20% | **保留** | 冻结风机不产电，FAN_SUM 无误差；CT 损耗基本可信 |
| **🔵 D 类** | 0 < P_freeze < 1 MW（任意 r_anom） | **保留** | 冻结功率极小（<线路容量 0.3%），损耗偏差可忽略 |
| **🟡 G 类** | P_freeze = 0 **且** 20% < r_anom ≤ 30% | **灰色/视时长决定** | CT 有中度异常，短时可保留，长时建议删除 |
| **🔴 C 类** | P_freeze = 0 **且** r_anom > 30% | **删除** | CT 严重失真，损耗不可信 |
| **🔴 F 类** | P_freeze > 5 MW（任意 r_anom） | **删除** | 冻结功率与线路损耗同量级，FAN_SUM 严重锁定偏差 |
| **🔴 E-删** | 1 ≤ P_freeze ≤ 5 MW **且** r_anom > 20% | **删除** | 中等冻结功率 + CT 也失真，双重不可信 |
| **🟢 E-留** | 1 ≤ P_freeze ≤ 5 MW **且** r_anom ≤ 20% | **保留** | 中等冻结功率但 CT 仍可信，净影响可接受 |

> **灰色地带（G 类）细化规则**：
> - 持续时长 ≤ 60 分钟 → 保留（短暂扰动）
> - 持续时长 > 8 小时 → 删除（可能是持续传感器故障）
> - 60 分钟 < 时长 ≤ 480 分钟 → 标记，建议与运维记录核对后决定

## 6. 特殊情形处理说明

### 6.1 WU "D 类"：固定冻结值 0.979 MW

WU 的 513 个 D 类分段中，**403 个分段的功率值精确为 0.979 MW**，这是同一台（或同组）风机长期处于特定工作点冻结的特征。对损耗建模影响：

- 0.979 MW ≈ 线路装机容量的 0.26%，在统计模型中属于噪声级别
- 这些分段的 WU 异常占比中位数为 **0.0**，说明 CT 在这些时段是正常的
- **建议：保留用于损耗建模，但在设备级别标记该台风机存在长期冻结问题**

### 6.2 DING "G 类"：零功率 + 异常20-30%（4.4% 占比）

DING 的 G 类偏高，根本原因是 DING CT 传感器存在系统性正偏，导致 L = FAN_SUM − CT 在正常情况下也偶发负值，触发异常判定。

**区分方法**：
- 若 CT 异常是因为 CT > FAN_SUM（负损耗），该分段的 CT 是问题，可考虑保留（标记）
- 若 CT 异常是因为损耗残差超出 2σ，说明 CT 与模型预测偏差大，需结合具体数值判断

### 6.3 长时段零功率分段（>8小时）

BING 有 128 个、DING 有 67 个、WU 有 126 个"持续>8小时、零功率、低异常（≤10%）"的分段，均属于 A 类（建议保留）。

## 7. 实施建议与操作步骤

### 7.1 清洗规则（代码可实现的判断逻辑）

对每条线路 `LINE`，为 `freeze_count_segments_v3.csv` 中的每一行打上决策标签：

```python
def tag_segment(row, line):
    P = row[f'{line}重复风机功率之和MW']
    R = row[f'{line}异常占比']
    T = row['持续时长(min)']
    
    if P > 5.0:
        return '删除-F'                          # 高功率冻结，必删
    elif 1.0 <= P <= 5.0:
        if R <= 0.20:
            return '保留-E'
        else:
            return '删除-E'                      # 中等功率 + CT 也异常
    elif 0.0 < P < 1.0:
        return '保留-D'                          # 极小功率，可忽略
    else:  # P == 0
        if R <= 0.20:
            return '保留-A'                      # 零功率低异常
        elif R <= 0.30:
            if T <= 60:
                return '保留-G-短'               # 短时灰色，保留
            elif T <= 480:
                return '待核实-G-中'             # 中等时长，人工核对
            else:
                return '删除-G-长'               # 长时CT问题，删除
        else:
            return '删除-C'                      # 零功率 + 高异常，删除
```

### 7.2 在 SCADA 数据层面的应用

1. **生成时间段排除列表**：对标签为 `删除-*` 的分段，记录其 `开始时间` 和 `结束时间`
2. **按线路分别处理**：BING 用 BING 的排除列表，DING 用 DING 的，WU 用 WU 的
3. **不跨线路影响**：一条线路的分段不影响另一条线路的可用数据
4. **重新构建 FAN_SUM**：删除时段仅删除该时段的该线路数据行，不影响其他线路

### 7.3 建议的清洗优先级

| 优先级 | 清洗内容 | 影响 |
|:---:|:---|:---:|
| 🔴 P1（立即执行） | WU F 类（45,669 min，占 11.2%） | 消除 WU 损耗模型最大污染源 |
| 🔴 P1 | BING/DING F 类（各约 2,317/614 min） | 清除短时高功率冻结 |
| 🟡 P2（次要） | 三线路 C 类（零功率 + 高异常） | 清除 CT 严重失真时段 |
| 🟡 P2 | 三线路 G-长（零功率 + 异常20-30% + 长时） | 清除持续性 CT 问题 |
| 🟢 P3（可选） | DING G 类结合 CT 绝对值核查 | DING CT 偏差专项处理 |
